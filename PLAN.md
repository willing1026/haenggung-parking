## 전략 기반

### 부족 정의
**"행궁동을 진짜 좋아하는 단골들"**

| 항목 | 내용 |
|------|------|
| 내면 서사 | "나는 행궁동을 남들보다 잘 아는 사람이다" |
| 부족 규범 | "우리는 행궁동 갈 때 미리 정보를 챙기고, 좋은 곳은 알려준다" |
| 위상 유형 | 동류집단 — "우리 다 같이 잘 즐기자" |
| 꿈 | 행궁동 갈 때마다 새로운 발견 / 나만 아는 좋은 곳 |
| 두려움 | 주차 때문에 망친 나들이 / 정보 없이 갔다가 헛걸음 |

### 변화 진술문
> "행궁동을 방문할 사람을 **'불확실한 탐색자'**에서 **'행궁동을 제대로 아는 사람'**으로 이동시킨다"

### 3문장 마케팅 약속
1. 이 사이트는 **행궁동을 제대로 즐기고 싶은 단골**을 위한 것이다.
2. 나는 **행궁동 방문할 때마다 더 잘 즐기고 싶어하는 사람**에 집중한다.
3. 이 사이트를 보면 당신은 **행궁동을 남들보다 잘 아는 사람**이 된다.

### 리마커블 기준
> "이걸 보고 카톡방에 공유할 이유가 있는가?"

## 콘텐츠 전략

### AI 콘텐츠 (씨앗) → 큐레이션 UGC (지금 바로) → 직접 UGC (Month 2+)

```
지금 (부족 0명)
└── AI가 씨앗 콘텐츠 생성 (주 2~3편)
    + SNS 큐레이션 즉시 시작 ("이번 주 베스트 포스트")

Month 1 후반 (구독자 50명)
└── 가장 활발한 방문자 3~5명 식별
    → "제보해줘" 첫 DM

Month 2 (구독자 150명)
└── 제보 폼 오픈 ("오늘의 행궁동 알려줘")
    → 제보 → 편집 → 발행 → 제보자 태그

Month 3+ (부족 문화 형성)
└── AI 팀 역할 = 큐레이터 + 편집자로 전환
```

## Month 1: 기반 공사 (2026-06-05 ~ 07-04)

**목표:** "새는 양동이 막기 + 씨앗 콘텐츠 + 애드센스 신청"

### Week 1 (Jun 5~11): 양동이 고치기

**SEO/배포 에이전트:**
- [ ] Privacy Policy 페이지 생성 (`/privacy`)
- [ ] About 페이지 생성 (`/about`)
- [ ] Contact 페이지 생성 (`/contact`)
- [ ] sitemap.xml 자동생성 설정 (`next-sitemap` 패키지)
- [ ] robots.txt 최적화
- [ ] OG 이미지 최적화 (SNS 공유 썸네일)

**허락 사다리 에이전트:**
- [ ] 카카오 채널 개설 또는 이메일 구독 폼 설계
- [ ] 사이트 내 구독 유도 배너 설계
- [ ] 구독 유도 카피 초안 3종 작성

**부족 인텔 에이전트:**
- [ ] 첫 번째 부족 세계관 브리프 작성
- [ ] `#행궁동` `#행리단길` 최근 7일 인기 포스트 분석
- [ ] 큐레이션 대상 3건 선별

**콘텐츠 에이전트:**
- [ ] 씨앗 콘텐츠 5편 초안 작성 → 팀장 검토 → 2~3편 발행
  - "행궁동 주차 완전정복 — 시간대별 꿀팁"
  - "행리단길 카페 단골이 고른 TOP 5 (이유 포함)"
  - "이번 주 행궁동 — 무슨 일 있었나 (2026-06-1주)"
- [ ] 각 글 하단에 "오늘 행궁동 어땠어요? 알려주세요" 제보 유도 문구 삽입

**팀장 (Opus):**
- [ ] 콘텐츠 리마커블 검토 (발행 승인)
- [ ] Week 1 완료 후 양동이 점검: "허락 사다리가 작동하는가?"

**Week 1 완료 기준:**
- Privacy Policy/About/Contact 라이브
- 구독 장치 1개 작동
- 콘텐츠 2편 이상 발행
- 애드센스 신청 준비 완료

## 자기수정 프로토콜

### 주간 (매주 월요일)
1. 애널리스트 주간 리포트 확인
2. 팀장 체크리스트 적용:
   - 재방문율 추세? (↑ 딥 / 평탄 막다른 길 점검)
   - 구독 전환율 추세?
   - 콘텐츠 공유 수 추세?
3. 전술 수정 사항 반영 → 해당 에이전트에게 업데이트

1. 부족 지표 vs 목표 달성률 전체 점검
2. "딥인가, 막다른 길인가?" 팀장 최종 판단
3. 다음 달 계획 수정 → PLAN.md 업데이트 (버전 올리기)
4. 전략 변경 시 팀 전체에 공유
